# Session 31 — 2026-04-27 — Structured CoT Grammar Exploration (reverted)

**Scope:** Full exploration of GBNF grammar-constrained Chain-of-Thought for B-KODE. Inspired by `andthattoo/structured-cot` (22× thinking token compression). Integrated, tested, and ultimately **reverted** — incompatible with agentic tool use.

### Root Cause: Why Grammar Failed

1. **llama-server rejects grammar+tools in same request** — hard limitation, documented in source: "Cannot use custom grammar constraints with tools"
2. **2-pass approach breaks** — grammar iteration stores assistant message with `reasoning_content`; next iteration with tools triggers "prefill incompatible with enable_thinking" (400)
3. **Without tools, agent hallucinates** — model writes pseudo tool calls as text instead of executing real ones
4. **Grammar sampler inhibited during reasoning** (fix #20970 in llama.cpp) — reasoning is free-form even with grammar, only content is constrained

### Key Findings

- GBNF grammar works perfectly for **non-agentic** generation (direct LLM calls without tools)
- The structured-cot paper tested on HumanEval+/LiveCodeBench — pure code generation, zero tool use
- Grammar reduces thinking tokens ~22× on supported benchmarks, but at 118 t/s on local RTX 4090 this is irrelevant — 25s vs 1.2s saving
- The `reasoning-format=deepseek` + grammar interaction is complex: grammar applies to raw output, server splits into reasoning/content channels
- `supports_grammar` detection pattern is valuable infrastructure for future provider capability checks

# Session 33 — 2026-04-27 — Agent Loop Stagnation Fixes

**Scope:** Fix critical regression from placeholder change + add loop detection to prevent infinite agent loops.

### Problem

Session `tui_c06a1979` (34 msgs) revealed two critical issues:

1. **Reasoning leaking into content** — When tools were active and `response_text` was empty, `reasoning_text[:200]` was stored as the assistant message content. The model saw its own thinking as "previous response" and got stuck repeating the same reasoning 15×.
2. **Infinite tool loop** — Agent read `ROADMAP.md` (first 10 lines) 10 consecutive times without progressing. Same tool + same args, never adapting.

### Fixes

| # | Fix | How |
|---|---|---|
| 1 | Placeholder | When `tool_calls_accum` is not empty → `[procesando herramientas...]`. Only fall back to reasoning text for final responses (no tools). |
| 2 | Loop detector | Tracks `(tool_name, args)` hash. After 3 consecutive identical calls → injects system message: "Has repetido la misma herramienta 3 veces sin progreso. Resume lo que sabes AHORA." + forces loop continue. |

### Design

```
Loop detector flow:
  same tool_key 3× → system msg → continue (skip tool execution)
  different tool_key → reset counter → execute normally
```

System message forces model to synthesize what it knows instead of calling the same tool again.

### Tests

110 passed — no regressions.

### What Worked

- **BytIA identity intact** — greeting in Spanish, kernel protocols active, clean final response.

### Remaining Edge Cases

- Bash pipe errors could give clearer feedback to accelerate tool switching
- Loop detector ready but untriggered — needs a pathological case to validate end-to-end

# Session 38 — 2026-04-30 — Session Metadata Persistence + v0.7.7 Closure

**Scope:** Cerrar acciones pendientes de v0.7.7 e implementar persistencia de `token_count` y `model` en sesiones SQLite. 144 tests pasando.

### Problema

Las sesiones persistidas nunca registraban qué modelo respondió (`model` vacío) ni cuántos tokens se consumieron (`token_count` = 0). El Agent loop no actualizaba estos campos tras cada turno.

### Implementación

**`SessionStore.update_metadata()`** — nuevo método que acepta kwargs filtrados (`model`, `token_count`) y hace UPDATE parcial. Solo se permiten campos conocidos para evitar SQL injection.

**Hook en `Agent.chat()`** — tras cada turno de chat exitoso (junto a `update_title`), se persisten:
- `model`: nombre real del provider client (`provider_client.model`)
- `token_count`: estimación total de tokens de la sesión (`_estimate_tokens()`)
